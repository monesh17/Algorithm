üîç Manacher's Algorithm

Finding the Longest Palindromic Substring in Linear Time

üìã What is Manacher's Algorithm?

Manacher's algorithm is an elegant solution to find the longest palindromic substring in O(n) time complexity. It's much more efficient than the naive O(n¬≤) or O(n¬≥) approaches.

‚ö° Time Complexity: O(n)
üíæ Space Complexity: O(n)

üîß How It Works

Algorithm Steps:

Step 1: Preprocessing

Transform the string to handle even/odd length palindromes uniformly:

Original: "babad"
Preprocessed: "^#b#a#b#a#d#$"

The '^' and '$' are sentinel characters to avoid bounds checking.

Step 2: Initialize Variables
  • P[i]: Array storing palindrome radius at each position
  • C: Center of the rightmost palindrome found so far
  • R: Right boundary of the rightmost palindrome
Step 3: Main Algorithm

For each position i:

  1. Find the mirror position of i with respect to center C
  2. Use previously computed palindrome information to initialize P[i]
  3. Try to expand the palindrome centered at i
  4. Update C and R if the palindrome extends beyond R
Step 4: Extract Result

Find the position with maximum P[i] value and map it back to the original string.

üíª Code Breakdown

Preprocessing Function

private char[] preprocess(String s) { StringBuilder sb = new StringBuilder(2 * s.length() + 3); sb.append('^'); // Start sentinel for (int i = 0; i < s.length(); i++) { sb.append('#').append(s.charAt(i)); // Insert '#' between chars } sb.append("#$"); // End with '#' and sentinel '$' return sb.toString().toCharArray(); }

Main Algorithm

for (int i = 1; i < m - 1; i++) { int mirror = 2 * C - i; // Mirror of i across center C if (i < R) { // If i is within right boundary P[i] = Math.min(R - i, P[mirror]); // Use symmetry } // Try to expand palindrome centered at i while (T[i + 1 + P[i]] == T[i - 1 - P[i]]) { P[i]++; } // Update center and right boundary if expanded beyond R if (i + P[i] > R) { C = i; R = i + P[i]; } // Track the best palindrome found if (P[i] > bestLen) { bestLen = P[i]; bestCenter = i; } }

üéØ Key Insights

1. Preprocessing Trick

By inserting '#' between characters, we make all palindromes have odd length, simplifying the algorithm.

2. Symmetry Exploitation

If we're inside a known palindrome, we can use the palindrome property to avoid redundant comparisons.

3. Linear Time Guarantee

Each character is examined at most twice: once when expanding and once as part of a larger palindrome. This guarantees O(n) time complexity.
